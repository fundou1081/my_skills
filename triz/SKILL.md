# TRIZ 发明问题解决技能

## 技能目标

帮助用户将工程/产品问题转化为结构化的TRIZ问题，利用矛盾矩阵、40发明原理、76标准解等工具，
系统性地找到创新解决方案。

## 工作流程

### 第1步：问题收集

收集以下信息（可通过对话直接收集，不需要运行脚本）：
1. 系统/产品是什么
2. 遇到的核心困难/问题
3. 已有尝试及失败原因
4. 约束条件（成本、时间、技术限制等）

### 第2步：矛盾识别

根据问题特征，识别**技术矛盾**或**物理矛盾**：

**技术矛盾** — 改善A时B会恶化：
- 改善的是什么参数？（速度↑、强度↑、成本↓...）
- 改善后什么会变差？（可靠性↓、复杂度↑、重量↑...）
- 将改善/恶化参数映射到39个工程参数

**物理矛盾** — 同一参数需要相反的两个值：
- 什么参数处于两难？（温度/尺寸/硬度/速度...）
- 正向需求是什么？
- 反向需求是什么？

> 如需交互式引导矛盾识别，运行：
> `python3 ~/.workbuddy/skills/triz/scripts/contradiction_wizard.py`

### 第3步：工具查询

**技术矛盾 → 查矛盾矩阵**：
```bash
python3 ~/.workbuddy/skills/triz/scripts/matrix_lookup.py 改善参数编号 恶化参数编号
```
输出：推荐原理编号列表（按优先级）

**物理矛盾 → 应用分离原理**：
- 空间分离：不同部位满足不同需求
- 时间分离：不同时段满足不同需求
- 条件分离：不同条件下满足不同需求
- 整体与部分分离：整体和部分差异化

**物场问题 → 查76标准解**：
见 `references/76_solutions.md`

### 第4步：原理应用

1. 查看推荐原理详情：`references/40_principles.md`
2. 逐一分析每个原理
3. 思考如何应用到当前问题
4. 组合多个原理以获得更强方案

### 第5步：方案输出

输出格式：
```
## 创新解决方案报告

### 问题摘要
[一句话描述核心问题]

### 矛盾分析
- 技术矛盾：参数A(#X) ↔ 参数B(#Y)
- 或物理矛盾：参数Z需要[正向需求]和[反向需求]

### 推荐原理
| 原理 | 应用思路 |
|------|----------|
| #X 原理解释 | 如何应用到这个问题 |

### 候选方案
1. [方案1] — 利用XX原理，[具体做法]
2. [方案2] — 利用YY原理，[具体做法]

### 方案评估
| 方案 | 优点 | 缺点 | 符合理想解程度 |
|------|------|------|---------------|
| 方案1 | | | ★★★★☆ |
```

## 关键原则

1. **矛盾优先**：TRIZ的核心是识别和解决矛盾，先明确矛盾再找原理
2. **组合创新**：不要只看排名第一的原理，前3-4个都值得认真思考
3. **跨行业迁移**：每个原理都在各行业验证过，跨领域应用是关键
4. **理想解方向**：始终思考"最小干预、最大效果"的方案
