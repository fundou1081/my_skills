---

## 一、TRIZ 是什么

**TRIZ**（俄语：Теория решения изобретательских задач）
= **发明问题解决理论**
- 创始人：根里奇·阿奇舒勒（Genrich Altshuller），苏联工程师
- 诞生：1946年，基于250万份专利分析
- 核心思想：**创新有规律可循**，看似不同的技术问题往往有相同的解决原理

## 三、推荐工作流（5步法）

### 步骤 1：问题定义（问题描述）

**目标**：将模糊的实际问题转化为结构化的问题描述

**操作**：
1. 描述系统的核心功能是什么
2. 当前遇到了什么困难/瓶颈
3. 这个问题为什么重要（业务价值）
4. 已经尝试过什么方案，为什么不行

**产出**：「问题陈述卡片」
```
系统: _______________
遇到的问题: _______________
已尝试方案: _______________
失败原因: _______________
```

---

### 步骤 2：矛盾识别（核心步骤）

**两种矛盾，任选其一或两者都做**：

#### 2A. 技术矛盾 (Technical Contradiction)

**定义**：改善参数A时，参数B会恶化

**操作**（使用 `scripts/contradiction_wizard.py`）：
1. 明确改善什么参数
2. 明确这个改善会导致什么恶化
3. 将改善/恶化参数映射到39个工程参数
4. 得到两个参数编号

**常见矛盾示例**：
| 改善 | 恶化 | 场景 |
|------|------|------|
| 速度(9) | 可靠性(27) | 高速 → 高磨损 → 可靠性下降 |
| 强度(14) | 重量(1) | 高强度 → 重 → 运输成本上升 |
| 自动化(38) | 复杂性(36) | 高自动化 → 系统复杂 → 维护难 |
| 生产率(39) | 能量浪费(22) | 高产 → 高能耗 → 成本上升 |

#### 2B. 物理矛盾 (Physical Contradiction)

**定义**：同一个参数需要同时满足两个相反的要求

**操作**：
1. 找到核心矛盾参数（如温度、硬度、尺寸等）
2. 明确正向需求（在某种条件下需要X）
3. 明确反向需求（在另一种条件下需要-X）

**四种分离原理**：
| 分离类型 | 问题特征 | 解决思路 |
|----------|----------|----------|
| 空间分离 | 不同部位/位置需要相反 | 分区差异化设计 |
| 时间分离 | 不同时间段需要相反 | 先后/交替执行 |
| 条件分离 | 不同条件下需要相反 | 条件触发/自适应 |
| 整体与部分分离 | 整体和部分需求相反 | 局部差异化 |

### 步骤 3：查工具找解法

#### 路径A：矛盾矩阵 + 40原理

**适用**：已识别技术矛盾

**操作**（使用 `scripts/matrix_lookup.py`）：
```
python3 scripts/matrix_lookup.py <改善参数编号> <恶化参数编号>
```

**示例**：
```
python3 scripts/matrix_lookup.py 9 27
# 输出：速度(9) vs 可靠性(27)
# 推荐原理：21, 35, 11, 28
```

**查看推荐的原理详解** → `references/40_principles.md`

#### 路径B：76标准解

**适用**：已识别物场模型问题，或测量/检测类问题

**操作**：
1. 构建物场模型
2. 判断问题类型
3. 查对应分类的解法

**查看标准解详情** → `references/76_solutions.md`

---

### 步骤 4：原理应用 + 方案设计

**关键技巧**：
1. **不要只看排名第一的原理** — 前3-4个都值得认真思考
2. **跨行业迁移** — 每个原理都曾在各行各业被验证过
3. **原理组合** — 多个原理叠加往往产生最强方案
4. **多想"如果...会怎样"** — 逆向思维是TRIZ的核心

**方案评估标准**：
- 是否消除了核心矛盾？
- 是否利用了系统资源？
- 是否符合理想解方向？

---

### 步骤 5：理想解评估

**理想解定义**：
> 系统本身不存在（问题自行解决），或只做最小的改变实现最大的改善

**评估问题**：
1. 这个方案是否接近理想解？
2. 是否有更简单的方案？
3. 方案对系统的其他部分有什么影响？
4. 这个方案是否可以推广到其他问题？

## 四、ARIZ 复杂问题算法（可选）

对于复杂问题，使用 ARIZ-85C 的9步结构：

| 步骤 | 内容 | 关键产出 |
|------|------|----------|
| 1 | 问题分析 | 微观问题定义 |
| 2 | 问题模型分析 | 操作区 + 可用资源 |
| 3 | 最终理想解(IFR) | 物理矛盾识别 |
| 4 | 利用外部资源 | 小矮人模型 |
| 5 | 利用知识库 | 标准解 + 效应库 |
| 6 | 改变/重新表述问题 | 超系统视角 |
| 7 | 分析消除物理矛盾 | 方案质量评估 |
| 8 | 利用已找到的方案 | 扩展应用 |
| 9 | 分析过程 | 经验总结 |

## 六、案例演示

### 案例：手机散热问题

**问题陈述**：手机运行高性能游戏时发热严重，影响用户体验

**步骤1 - 矛盾识别**：
- 技术矛盾：提高性能(速度↑) → 温度升高(17↑)
- 改善参数：9（速度）
- 恶化参数：17（温度）

**步骤2 - 查矩阵**：
```
python3 scripts/matrix_lookup.py 9 17
```
→ 推荐原理：22, 21, 35, 2

**步骤3 - 原理应用**：
- 原理22「变害为利」：利用废热发电（续航补偿）
- 原理21「高速通过」：快速散热通道，减少热积累
- 原理35「参数变化」：改变散热材料属性（相变材料）
- 原理2「抽取分离」：将热源与敏感元件分离

**步骤4 - 方案**：
1. 石墨烯散热片（增大散热面积）
2. 相变储能材料（吸收峰值热量）
3. 热管将热量导向大面积金属边框（热扩散）
4. 智能降频保护机制（在温度临界时自动降频）

**评估**：综合方案符合理想解方向——最小干预、最大效果
