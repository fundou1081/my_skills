# 02. 结构变奏 (Structural Variation)

> **子功能**: 改编歌曲的结构 (段落顺序/重复/加段/速度), 歌词基本不变。

## 触发条件
- 用户说"加快" / "放慢" / "加桥段" / "分段" / "重复副歌"
- 用户说"改成 ___ BPM" / "加 D 段"
- 跟 01-style-conversion 叠加使用

## 5 种变奏模式

| 模式 | 适用 | 难度 |
|---|---|---|
| **加快** (Tempo Up) | 抒情 → 舞曲 | 低 |
| **放慢** (Tempo Down) | 快歌 → 抒情 | 低 |
| **加桥** (Add Bridge) | 增强叙事 | 中 |
| **分段** (Section Split) | 长副歌 → 拆两段 | 中 |
| **重复副歌** (Chorus Repeat) | 强化记忆 | 低 |

详见: [patterns.md](patterns.md)

## 工作流

1. 读原曲结构 (A1→B→C→A2→B2→C2→D→C3 之类)
2. 选 1-N 种变奏
3. 应用变奏规则
4. 重新生成 minimax 段落标记

## 跟 01 配合

```
原曲 + (01 曲风转换) + (02 结构变奏) → 新版
```

可以单独用 02, 也可以跟 01 叠加。

## 输出

- 新的段落标记 (例如: [verse] [chorus] [chorus] [chorus])
- 微调后的歌词 (适配新结构)
- minimax-music 输入
