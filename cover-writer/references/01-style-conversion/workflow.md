# 曲风转换工作流

## 4 步流程

### Step 1: 原曲画像
- 读原曲的 8 维度画像 (从 lyric-writer 借调框架)
- 提取: 原风格 / 主题 / 情感 / 句式 / 押韵 / 意象
- 输出: 原曲"风格 DNA"

### Step 2: 目标风格映射
- 用户给目标风格
- 查 [style-mapping.md](style-mapping.md) 的"风格转换矩阵"
- 找映射规则: 配器 / 节奏 / 调性 / 人声

### Step 3: description 转换
原 description (例如): "古风, 笛箫配乐, 女声, 慢板"
→ 改后 (例如): "古风元素 + EDM, 强劲 drop, 男女合唱, 快板"

转换原则:
- **保留核心情感** (例如: 思念 / 孤独)
- **改配器** (笛箫 → 合成器)
- **改节奏** (慢板 → 快板)
- **可加细节** (古筝 + 808 bass)

### Step 4: 歌词微调
- **歌词基本不变**
- 微调规则:
  - 加快 → 删冗余句, 强化 hook
  - 慢化 → 加抒情, 拆长句
  - 重复副歌 → 复制副歌段
  - 加桥 → 插入 D 段 (4-6 句)

## 不触发
- 用户没指定目标风格 → 问"想改编成什么风格?"
- 用户要改歌词 (不是改编曲) → 转给 lyric-writer

## 关键原则

> **风格是配器 + 节奏 + 调性 + 配器, 不是歌词。**  
> 改编曲 = 改这些, 歌词不动。
