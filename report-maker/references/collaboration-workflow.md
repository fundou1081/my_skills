# 报告协作与版本管理指南

## 版本号规则

### 语义化版本

```
v{主版本}.{次版本}.{修订版本}-{状态}

示例：
v1.0.0-draft    # 初稿
v1.0.0-review  # 评审中
v1.0.0         # 正式发布
v1.1.0         # 次版本更新（小功能/内容补充）
v2.0.0         # 主版本更新（重大结构调整）
v1.0.1         # 修订版（错误修正）
```

### 版本状态流转

```
draft → in-review → approved → published
  ↓         ↓
archived  rejected → draft（修订后重新评审）
```

## 协作流程

### 1. 创建阶段

```
负责人：报告发起人
任务：
├── 确定报告主题和目标
├── 选择分析框架
├── 确定章节结构
└── 创建初始版本 (v0.1-draft)
```

### 2. 撰写阶段

```
负责人：内容撰写者
任务：
├── 按章节分工撰写
├── 实时更新章节状态
├── 记录撰写过程中的问题
└── 标记章节完成
```

### 3. 内部评审

```
负责人：团队内部评审者
任务：
├── 检查内容的准确性和完整性
├── 补充遗漏信息
├── 修正错误表述
└── 添加评审意见
```

### 4. 外部评审

```
负责人：领域专家/利益相关方
任务：
├── 验证分析结论
├── 提供专业反馈
├── 指出潜在风险
└── 批准或提出修改建议
```

### 5. 发布与归档

```
负责人：报告负责人
任务：
├── 整合最终版本
├── 生成变更日志
├── 发布报告
└── 归档历史版本
```

## 评审检查清单

### 内容检查

- [ ] 报告目标与实际内容一致
- [ ] 数据来源可靠、引用规范
- [ ] 逻辑推理清晰、论证充分
- [ ] 结论有数据/案例支撑
- [ ] 无事实错误或过时信息

### 结构检查

- [ ] 标题准确反映内容
- [ ] 章节逻辑清晰、层次分明
- [ ] 摘要概括核心要点
- [ ] 图表清晰、标注完整
- [ ] 附录资料完整

### 受众适配检查

- [ ] 语言风格适合目标读者
- [ ] 专业术语使用恰当
- [ ] 信息密度适合阅读习惯
- [ ] 关键信息突出明显

## 协作工具推荐

| 工具类型 | 推荐选项 | 用途 |
|----------|----------|------|
| 协作文档 | Notion、飞书文档、腾讯文档 | 多人实时编辑 |
| 版本控制 | Git (GitHub/GitLab) | 代码式文档管理 |
| 评审工具 | GitHub PR、Pull Request | 结构化评审流程 |
| 任务管理 | Jira、Todoist、飞书任务 | 任务分配追踪 |
| 状态同步 | Slack、飞书群 | 实时沟通 |
