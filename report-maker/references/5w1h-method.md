# 5W1H 分析方法详解

## 概述

5W1H 是一种经典的系统性分析方法，通过六个维度全面审视问题。

## 六个维度

### 1. What（什么）

**核心问题**：
- 这是什么问题/现象/事物？
- 主要特征是什么？
- 涉及哪些要素？

**应用场景**：
- 问题界定
- 现象描述
- 对象定义

**示例问题**：
```
- 这个项目的核心交付物是什么？
- 问题的主要表现形式是什么？
- 关键成功要素是什么？
```

---

### 2. Why（为什么）

**核心问题**：
- 为什么会出现这个问题？
- 为什么需要解决？
- 根本原因是什么？

**应用场景**：
- 原因分析
- 动机探索
- 价值论证

**示例问题**：
```
- 为什么销售额下降？
- 为什么用户流失？
- 为什么需要这个功能？
```

**分析方法**：
- 鱼骨图
- 因果链分析
- 价值链分析

---

### 3. Who（谁）

**核心问题**：
- 涉及哪些人/组织？
- 谁是利益相关者？
- 谁负责？谁执行？谁受益？

**应用场景**：
- 利益相关方分析
- 责任划分
- 用户画像

**示例问题**：
```
- 目标用户是谁？
- 谁是决策者？
- 谁会受到影响？
```

**分析工具**：
- 利益相关方矩阵
- RACI 矩阵
- 用户角色地图

---

### 4. When（何时）

**核心问题**：
- 何时发生？
- 时间节点是什么？
- 周期性特征？

**应用场景**：
- 时间线分析
- 阶段划分
- 趋势判断

**示例问题**：
```
- 项目里程碑是什么时候？
- 问题何时开始出现？
- 最佳实施时机？
```

**分析工具**：
- 时间轴图
- 甘特图
- 趋势图

---

### 5. Where（何地）

**核心问题**：
- 在哪里发生？
- 地域/空间特征？
- 环境因素？

**应用场景**：
- 区域分析
- 环境扫描
- 空间布局

**示例问题**：
```
- 目标市场在哪里？
- 问题集中在哪些区域？
- 线上还是线下？
```

**分析工具**：
- 地理分布图
- 空间矩阵
- 环境分析模型

---

### 6. How（如何）

**核心问题**：
- 如何实施？
- 如何解决？
- 方法/路径是什么？

**应用场景**：
- 方案设计
- 流程优化
- 实施路径

**示例问题**：
```
- 如何达成目标？
- 如何解决问题？
- 如何衡量效果？
```

**分析工具**：
- 流程图
- 路线图
- 实施计划表

---

## 组合使用模式

### 完整分析模式

```
1. What → 定义问题
2. Why → 分析原因
3. Who → 确定角色
4. When → 把握时机
5. Where → 明确范围
6. How → 设计方案
```

### 问题诊断模式

```
What → Why → How
（现象 → 原因 → 方案）
```

### 项目规划模式

```
What → Who → When → Where → How
（目标 → 团队 → 时间 → 地点 → 方法）
```

---

## 与其他方法结合

### 5W1H + 5Why

```
What: 问题描述
  └─ Why (第1层)
      └─ Why (第2层)
          └─ Why (第3层)
              └─ Why (第4层)
                  └─ Why (第5层) → 根本原因
How: 解决方案
```

### 5W1H + SWOT

```
Where + When → 外部环境（O/T）
Who + What → 内部条件（S/W）
Why + How → 战略选择
```
