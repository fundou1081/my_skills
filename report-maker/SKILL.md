# 专业报告制作技能

## 核心能力

本技能帮助用户系统化地制作专业报告，核心特点：

- **智能框架生成**：通过 `generate_analysis_framework.py` 脚本自动生成分析待办清单
- **深度追问机制**：每个维度包含「基础追问 → 追问补充 → 深度追问」三层递进
- **广度覆盖保证**：5W1H 六维度 + 多专家视角，确保分析无死角
- **可定制参数**：调研深度(L1-L4)、目标受众、报告篇幅

## 工作流程

### 第一步：需求确认（必做）

在开始制作报告前，必须向用户确认以下参数：

#### 1. 分析方法选择
询问用户希望使用哪种分析方法（可多选）：
- **5W1H**：What、Why、Who、When、Where、How（广度探索）
- **5Why**：连续追问5次"为什么"，挖掘根本原因（深度探索）
- **5问法**：从问题定义到效果衡量的系统分析
- **7问法**：5问 + 风险分析 + 资源规划
- **多专家视角**：邀请3-7位专家从不同角度审视

#### 2. 调研深度
让用户选择调研深度等级：
- **L1-快速概览**：快速扫描，核心要点，适合初步了解
- **L2-标准调研**：常规深度，覆盖主要维度
- **L3-深度调研**：详尽分析，多角度论证（含5Why追问）
- **L4-专家级**：学术级别，引经据典，全面详实（含7问完整框架）

#### 3. 目标受众
指定报告的阅读对象：
- **一般大众**：通俗易懂，避免专业术语，图表辅助
- **专业人士**：行业标准术语，数据支撑
- **管理层/决策者**：简洁有力，突出结论和建议
- **学术研究**：严谨规范，引用文献
- **自定义**：用户指定特定受众

#### 4. 报告篇幅
让用户选择篇幅控制方式（四选一或多选）：
- **按页数**：指定总页数（如"10页"）
- **按章节**：指定章节数量（如"5个章节"）
- **按字数**：指定总字数（如"5000字"）
- **按时间**：根据可用时间动态调整（如"30分钟能读完"）

#### 5. 报告主题
明确报告的核心主题和具体问题。

### 第二步：生成分析框架待办清单（自动执行）

使用框架生成器脚本生成系统性的追问清单：

```bash
python3 scripts/generate_analysis_framework.py "<主题>" \
    --depth-level <L1/L2/L3/L4> \
    --domain <技术/商业/产品/管理/政策/通用> \
    --format markdown
```

#### 框架生成器支持的维度

| 维度 | 说明 | 适用场景 |
|------|------|----------|
| 5W1H 六维度 | What/Why/Who/When/Where/How | 全面了解问题 |
| 5Why 追问链 | 5层递进追问 | 挖掘根本原因 |
| 5问/7问 | 系统性解决方案设计 | 形成完整方案 |
| 多专家视角 | 3-7位角色分析 | 多角度审视 |

#### 生成的待办清单结构

每个维度包含三层追问：
1. **基础追问** - 核心问题
2. **追问补充** - 2-3个递进问题
3. **深度追问** - 🔍 挖掘更深层的信息

#### 专家视角生成规则

| 领域 | 专家组合 |
|------|----------|
| 技术 | 技术架构师、安全专家、性能工程师、运维专家、数据工程师 |
| 商业 | 战略顾问、财务分析师、市场研究员、运营专家、风险控制官 |
| 产品 | 产品经理、用户体验设计师、数据分析师、行业专家、竞品分析师 |
| 管理 | 项目经理、HR专家、变革管理专家、流程优化专家、领导力教练 |
| 政策 | 政策分析师、法律顾问、经济学家、社会学家、行业监管专家 |
| 通用 | 行业专家、数据分析师、创新顾问、执行专家、评审专家 |

### 第三步：系统性执行追问（按清单进行）

#### 广度探索：5W1H 执行

```
Phase 1: 广度探索 - 5W1H 六维度分析

每个维度按三层递进执行：

☐ What（什么）
   基础追问 → 追问补充 → 🔍 深度追问

☐ Why（为什么）
   基础追问 → 追问补充 → 🔍 深度追问

...（其余维度类似）
```

#### 深度探索：5Why 执行

```
Phase 2: 深度探索 - 5Why 追问链条

☐ 第1层追问
   追问 + 回答格式【原因陈述 → 证据支撑】
   ⚠️ 若无法回答 → 标记需要更多信息

☐ 第2层追问 → ... → 第5层追问
   直到找到根本原因
```

#### 系统探索：5问/7问 执行

```
Phase 3: 系统探索 - 5问法核心框架

☐ 问题1：本question → 子问题 → ✅ 完成标准
☐ 问题2：... → ... → ✅ 完成标准
...（5问全部完成或7问全部完成）
```

#### 多视角审视：专家分析

```
Phase 4: 多视角审视 - 专家角色分析

☐ 【行业专家】
   关注问题 → 期望输出：3-5条洞察 + 1-2条建议

☐ 【数据分析师】
   ...（至少完成3位专家）
```

### 第四步：信息整合与报告撰写

#### 报告结构模板

```markdown
# [报告标题]

## 摘要/执行概要
- 核心结论（3-5条）
- 关键建议
- 适用范围

## 一、背景与问题界定
- 问题背景
- 研究范围
- 分析方法说明

## 二、深度探索发现
### 2.1 5W1H 六维度分析
### 2.2 根本原因分析（5Why）
### 2.3 系统性方案设计（5问/7问）

## 三、多视角综合分析
### 3.1 [专家1] 视角
### 3.2 [专家2] 视角
### 3.3 [专家3] 视角

## 四、关键发现
- 发现1：[标题]
  - 详细说明
  - 数据支撑
- 发现2：...

## 五、建议与方案
### 5.1 短期建议
### 5.2 中期建议
### 5.3 长期建议

## 六、风险与局限
- 分析局限
- 潜在风险
- 不确定性因素

## 七、附录（可选）
- 参考文献
- 数据来源
- 术语说明
```

#### 受众适配指南

| 受众类型 | 语言风格 | 内容侧重 | 格式建议 |
|---------|---------|---------|---------|
| 一般大众 | 通俗易懂，类比说明 | 结论优先，故事化 | 图表为主，文字精简 |
| 专业人士 | 行业术语，精准表达 | 方法论，数据支撑 | 结构清晰，引用规范 |
| 管理层 | 简洁有力，结论导向 | 决策信息，ROI分析 | 执行摘要+关键图表 |
| 学术研究 | 严谨规范，逻辑严密 | 理论框架，文献综述 | 引用完整，论证详实 |

## 协作与版本管理

### 协作流程

报告制作支持多人协作：

```
📝 撰写阶段 → 🔍 内部评审 → 👥 外部评审 → ✅ 发布
     ↓              ↓            ↓
  分配章节      检查准确性    领域专家审核
```

#### 协作角色

| 角色 | 职责 |
|------|------|
| 报告发起人 | 确定主题、框架、目标受众 |
| 章节撰写者 | 分工撰写各章节内容 |
| 内部评审 | 团队内部检查准确性和完整性 |
| 外部评审 | 领域专家验证结论和建议 |
| 报告负责人 | 最终审核、发布、归档 |

### 版本管理

#### 版本号规则

```
v{主版本}.{次版本}.{修订版本}-{状态}

示例：
v0.1-draft      # 初稿
v0.2-draft      # 修订稿
v1.0.0-review   # 评审中
v1.0.0          # 正式发布
v1.1.0          # 小更新
v2.0.0          # 大版本更新
```

#### 版本状态流转

```
draft → in-review → approved → published
  ↓         ↓
archived  rejected（修订后重新评审）
```

### 版本管理工具

使用 `report_version_manager.py` 脚本管理版本：

```bash
# 创建新报告
python3 scripts/report_version_manager.py create --name "2026年市场分析报告"

# 更新章节
python3 scripts/report_version_manager.py update --section "摘要" --content "..."

# 标记章节完成
python3 scripts/report_version_manager.py complete --section "摘要"

# 创建新版本
python3 scripts/report_version_manager.py version --version "v1.0.0" \
    --changelog "完成初稿" "补充数据" --author "张三"

# 查看状态
python3 scripts/report_version_manager.py status

# 添加评审人
python3 scripts/report_version_manager.py review --reviewer "李四" \
    --version "v1.0.0" --comment "请检查第三章数据"

# 评审批准
python3 scripts/report_version_manager.py review --reviewer "李四" \
    --approve --comment "数据准确，结构清晰"

# 查看变更日志
python3 scripts/report_version_manager.py changelog -o CHANGELOG.md

# 生成报告草稿
python3 scripts/report_version_manager.py draft -o report_draft.md
```

## 注意事项

1. **参数必确认**：不要跳过需求确认步骤直接开始写作
2. **框架自动生成**：使用脚本生成分析框架，确保追问的系统性
3. **深度要递进**：每个问题至少追问2层，不要只问表面问题
4. **广度要覆盖**：5W1H 六维度至少完成基础问答
5. **专家要多元**：至少3位不同背景的专家视角
6. **受众要适配**：同一内容，不同受众的表达方式差异很大
7. **篇幅要控制**：在开始写作前规划好各章节的篇幅分配
8. **版本要追踪**：重要更新记得创建新版本，记录变更日志
9. **评审要认真**：每个版本发布前必须经过评审

## 框架生成器使用指南

### 基本用法

```bash
# 生成完整框架（L3深度）
python3 scripts/generate_analysis_framework.py "电商平台用户体验优化" --depth-level L3

# 仅生成5W1H分析
python3 scripts/generate_analysis_framework.py "微服务架构选型" --framework 5w1h

# 生成7问完整框架
python3 scripts/generate_analysis_framework.py "组织变革方案" --framework 7q --domain 管理

# 指定领域获取更专业的专家组合
python3 scripts/generate_analysis_framework.py "AI落地策略" --domain 技术 --depth-level L4

# 输出到文件
python3 scripts/generate_analysis_framework.py "主题" -o ./analysis_framework.md
```

### 参数说明

| 参数 | 说明 | 可选值 |
|------|------|--------|
| topic | 报告主题 | 必填 |
| --framework, -f | 分析框架类型 | full, 5w1h, 5why, 5q, 7q |
| --depth-level, -d | 调研深度级别 | L1, L2, L3, L4 |
| --domain | 领域类型 | 技术, 商业, 产品, 管理, 政策, 通用 |
| --initial-problem, -i | 5Why的初始问题 | 字符串 |
| --output, -o | 输出文件路径 | 文件路径 |
| --format | 输出格式 | markdown, json |
