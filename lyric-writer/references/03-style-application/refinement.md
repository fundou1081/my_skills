# 调整机制 (Refinement)

歌词生成后, 用户常要改。提供 5 种调整模式 + 1 个回退机制。

## 5 种调整模式

### 模式 1: 句级调整 (Sentence-level)
"第 3 句改成..."
"副歌第 2 句太长了, 缩短"

**实现**: 直接重写指定句, 保持其他不变。

### 模式 2: 意象替换 (Imagery Swap)
"潮 改成 风"
"灯 改成 月"

**实现**: 全局替换, 但要检查:
- 韵脚是否还押?
- 情感是否一致?
- 是否破坏矛盾修辞?

### 模式 3: 调性调整 (Tone Shift)
"更现代点" / "更文言点" / "更直白点" / "更含蓄点"

**实现**: 调整 5 维度的"表达方式" + "特殊技法" 字段, 重新生成。

### 模式 4: 结构重排 (Structure Reorder)
"不要桥段"
"A1 改 A2 的内容"
"加一段 pre-chorus"

**实现**: 重新规划段落结构, 内容做相应迁移。

### 模式 5: 完全重写 (Full Rewrite)
"完全不要这版, 重写"
"按另一个方向写"

**实现**: 保留主题和风格, 重新跑 3 阶段流程。

## 回退机制 (Rollback)

每次调整生成新版本时, **保留历史**:

```
v1 (initial draft)
v2 (sentence-level fix)
v3 (imagery swap)
v4 (full rewrite)
...
```

最多保留 5 轮, 超出时给出"恢复到 v?" 选项, 用户选一个回到那版。

## 调整超限处理

如果用户调整超过 3 轮仍不满意, **主动建议回到 01 重新对画像**:

> "我们已经调整 3 轮, 还没到满意。  
> 是不是画像 (8 维度) 哪里有偏差?  
> 要不要回到 [01-style-extraction](../01-style-extraction/README.md) 重新对一下?"

## 实现要点

- 每次调整后, **重新跑自检清单** (避免越改越差)
- **不要死守原版本**, 用户要改就改
- **调整建议可叠加** (例如"句级 + 调性级" 同时)

## 用户友好提示

```
我帮你做以下调整:
- 模式 1: 改第 3 句为 "..."
- 模式 2: 全局把 "灯" 换成 "月"

确认? 还是只做某一项?
```
