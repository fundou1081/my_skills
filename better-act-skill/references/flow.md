## 范例 1: 显式触发 + TIDD-EC 框架

**场景**: 用户用 TIDD-EC 框架引导自己做一个 skill。

---

**👤 用户**: 用 TIDD-EC 帮我设计一个"提示词框架" skill, 让用户先用框架理清需求再开工。

**🤖 AI** (Step 2 - Choose):

> 好的, 你的任务听起来是设计一个**结构化、精确**的 skill, **TIDD-EC 框架很合适**。  
> 我们一步步来, 先问最关键的:
>
> **1. T (任务)** — 这个 skill 用来做什么? 一句话说清。  
> 例: "帮用户在动手前用提示词框架理清需求"

**👤 用户**: 帮用户在动手前用提示词框架 (TIDD-EC 等) 理清真实需求, 再正式开工。

**🤖 AI**:

> 明白。下一个问题:
>
> **2. I (指令)** — 这个 skill 内部要做什么?  
> 给我 3-5 步大致流程, 我参考。

**👤 用户**: 检测信息不足 → 选框架 → 引导提问 → 需求澄清表 → 生成 prompt。

**🤖 AI**:

> 清晰。**3. D (细节)** —
> - 目标受众: 谁会用这个 skill? (开发者 / 产品经理 / 所有人)
> - 语气: 顾问 / 教练 / 朋友
> - 触发方式: 显式 ("用 X 框架") / 隐式 (AI 主动判断)

**👤 用户**: 所有人; 顾问; 两者都支持。

**🤖 AI**:

> **4. D (资料)** — 5 个内置框架是哪些?  
> 已经有完整的: TIDD-EC, ICDO, CRISPE, BROKE, CO-STAR? 还是更多?

**👤 用户**: 这 5 个就够, 留扩展接口。

**🤖 AI**:

> **5. E (范例)** — 给我一段"理想对话"片段, 我按这个风格学。

**👤 用户**: 暂无, 你看着办。

**🤖 AI** (Step 4 - Clarify):

> 好, 我先输出**需求澄清表**让你确认:

```
# 需求澄清表 (TIDD-EC 框架)

| 要素 | 内容 |
|---|---|
| T (任务) | 帮用户在动手前用提示词框架理清真实需求 |
| I (指令) | 检测→选框架→引导→澄清表→生成 prompt |
| D (细节) | 受众=所有人; 语气=顾问; 触发=显式+隐式 |
| D (资料) | 5 个内置框架 (TIDD-EC/ICDO/CRISPE/BROKE/CO-STAR), 留扩展接口 |
| E (范例) | 暂无, 按 sv-trace 风格 |
| C (限制) | 符合 AgentSkills 规范, name 小写连字符, description 必填 |
```

> 确认? 还是改某几项?

**👤 用户**: 确认。

**🤖 AI** (Step 5 - Generate):

> 好, 我现在按这个表生成完整 skill prompt, 然后开始写文件。

## 范例 2: 隐式触发 + AI 自动选框架

**场景**: 用户只说了一句"帮我做一个 skill", AI 主动触发。

---

**👤 用户**: 帮我做一个 skill。

**🤖 AI** (Step 1 - Detect):

> 你好! 这个任务比较模糊 (没说做什么 skill、给谁用、输出什么), 我先帮你理清。  
>  
> 你的任务对准确性敏感 (是结构化输出), 我推荐用 **TIDD-EC 框架**。可以改。

**👤 用户**: 行。

**🤖 AI** (Step 3 - Guide):

> **1. T (任务)** — 这个 skill 用来做什么?

**👤 用户**: 帮开发者快速理解陌生代码库。

**🤖 AI**:

> 明白。**2. I (指令)** — 给 3-5 步大致流程。

**👤 用户**: 输入项目路径 → 扫描代码结构 → 生成概览文档。

**🤖 AI**:

> **3. D (细节)** — 目标受众? 风格?

**👤 用户**: 团队新成员; 简洁。

**🤖 AI**:

> **4. C (限制)** — 有什么硬要求? (支持语言 / 文档长度 / 格式)

**👤 用户**: Python/SV/Go; Markdown; 1 页。

**🤖 AI** (Step 4 - Clarify):

```
# 需求澄清表 (TIDD-EC)

| 要素 | 内容 |
|---|---|
| T | 帮开发者快速理解陌生代码库 |
| I | 输入路径 → 扫描结构 → 生成概览 |
| D | 受众=团队新成员; 风格=简洁 |
| D | (未提供) |
| E | (未提供) |
| C | Python/SV/Go; Markdown; 1 页 |
```

> D (资料) 和 E (范例) 缺, 不强求, 你可以补也可以不补。  
> 确认这个表?

## 范例 5: 复杂任务 + CRISPE

**场景**: 用户自己说"我也没想清楚"。

---

**👤 用户**: 帮我规划下个季度的工作, 我自己也没想清楚。

**🤖 AI** (Step 2):

> 收到, 这种"边想边理" 的任务推荐用 **CRISPE 框架**, 它允许 AI 在过程中反问你。  
>  
> CRISPE 6 要素: C (角色) / R (洞察) / S (意图) / P (风格) / E (实验=反问) / O (输出)  
>  
> 先问最关键的:
>
> **1. 角色** — 我应该扮演什么身份? (你的上级 / 教练 / 顾问)

**👤 用户**: 教练吧, 帮我理思路。

**🤖 AI**:

> **2. 洞察** — 给我一些背景: 你目前手头有哪些项目? 团队多大?

**👤 用户**: 3 个项目, 团队 5 人。

**🤖 AI** (E - Experiment 启动):

> **3. 实验模式开启** — 我接下来会一边反问你一边给建议, 你回答就行。  
>  
> 先问关键: **这 3 个项目哪个最关键? 哪个可以放缓?**

**👤 用户**: A 最关键, B 可以放缓, C 还在调研阶段。

**🤖 AI**:

> 明白。那我建议:  
> - A 项目: 重点投入 50% 资源  
> - B 项目: 维持 20% 资源  
> - C 项目: 10% 调研, 不投入开发  
> - 留 20% buffer  
>  
> 这个分配合理吗? 如果合理, 我再细化成季度 OKR。

## 范例 6: 营销文案 + CO-STAR

**场景**: 用户写公众号文章。

---

**👤 用户**: 帮我写一篇推广我们新产品 (AI 学习助手) 的公众号文章。

**🤖 AI** (Step 2):

> 好的, 这是面向受众的内容, 推荐用 **CO-STAR**。  
>  
> **1. C (背景)** — 产品是什么? 目标用户? 卖点?

**👤 用户**: AI 学习助手, 主打"边学边练"; Z 世代大学生; 卖点是"个性化学习路径"。

**🤖 AI**:

> **2. O (目标)** — 这篇文章想达成什么? (拉新 / 激活 / 留存)

**👤 用户**: 拉新, 引导预约。

**🤖 AI**:

> **3. S (风格)** + **4. T (语调)** — 风格活泼还是专业? 情感基调?

**👤 用户**: 活泼, 像朋友推荐。

**🤖 AI**:

> **5. A (受众)** — 18-25 岁大学生, 我假设了, 对吗?

**👤 用户**: 对。

**🤖 AI**:

> **6. R (回复格式)** — 标题、开头、结构、字数?

**👤 用户**: 标题抓眼球, 开头讲故事, 中间 3 个卖点, 结尾 CTA。500 字内。

**🤖 AI** (Step 4):

```
# 需求澄清表 (CO-STAR)

| 要素 | 内容 |
|---|---|
| C | AI 学习助手, 边学边练, 个性化路径, Z 世代大学生 |
| O | 拉新, 引导预约 |
| S | 活泼 |
| T | 朋友推荐 |
| A | 18-25 岁大学生 |
| R | 标题 + 故事开头 + 3 卖点 + CTA, 500 字内 |
```

> 确认?

## 提问节奏指南

### 黄金原则: 每次 1-2 个问题

**❌ 错误** (一次问 5 个):
```
"请告诉我:
1. 你的任务是什么?
2. 目标受众是谁?
3. 风格有什么偏好?
4. 有什么约束?
5. 输出格式?"
```

**✅ 正确** (一次 1-2 个):
```
"你的任务对准确性敏感, 推荐 TIDD-EC。  
先问最关键的: **1. T (任务)** — 这个 skill 用来做什么?"
```

### 提问顺序 (按 TIDD-EC)

1. **T (任务)** — 第一问 (最重要)
2. **I (指令)** — 第二问 (流程)
3. **D (细节)** — 第三问 (受众/语气)
4. **D (资料)** — 第四问 (有没有原始材料)
5. **E (范例)** — 第五问 (可选, 用户可以跳)
6. **C (限制)** — 最后问 (硬约束)

### 中途跳过

如果用户说:
- "资料暂时没有" → 跳过 D (资料)
- "没有范例" → 跳过 E (范例), 标记为 (未提供)
- "你自己看着办" → AI 填默认值, 标"AI 默认"

### 强制收口

**最多问 6 轮** (6 个要素), 第 6 轮后必须输出需求澄清表让用户确认, 不能无限问下去。

### 用户已经说过的, 不重复问

如果用户在 T 里已经说了风格 ("帮团队新成员做, 风格要简洁"), D (细节) 就不用再问风格, 直接问其他字段。
