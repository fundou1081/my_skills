## 显式触发 (用户说)

**用户说以下任一话时, 必须触发**:

### 强触发 (一听到就触发)
- "用 TIDD-EC 帮我..."
- "用 ICDO 帮我..."
- "用 CRISPE 引导我..."
- "用 BROKE 框架..."
- "用 CO-STAR 帮我..."
- "先帮我理清需求"
- "帮我分析该用哪个框架"
- "我想做 X, 但说不清楚"
- "我想做 X, 不知道从哪开始"
- "按 X 框架引导我"

### 中触发 (结合上下文判断)
- "理清需求再开始" → 触发
- "先想清楚再干" → 触发
- "帮我梳理一下" → 看上下文
- "想做个 X" → 看是否含完整 5W1H

### 弱触发 (看上下文)
- "做 X" → 短任务 + 信息不全 → 触发
- "帮个忙" → 通常不触发, 问清楚即可

## 隐式触发 (AI 主动判断)

**满足以下任意 2 条即触发** (默认阈值, 可调):

### 信号 1: 用户是**第一次**对话
- 无历史上下文
- 自我介绍 (新会话)

### 信号 2: 任务只有动词, 缺关键要素
- "帮我做 X" (没目标/受众/约束)
- "做个东西" (完全模糊)
- "写个文档" (写什么? 给谁看?)

### 信号 3: 描述长度 < 30 字
- 短请求通常信息不足
- 例外: "把下面这段话翻译成中文" (短但清楚)

### 信号 4: 没有约束条件
- 没说字数限制
- 没说格式要求
- 没说风格/语调
- 没说受众

### 信号 5: 用户说"随便"/"你看着办"/"都行"
- 强信号: 用户没想清楚
- **直接进入 BROKE 框架** (允许迭代)

### 信号 6: 涉及 3+ 领域
- "写代码 + 写文档 + 写测试 + 部署" → 复杂
- "做产品规划 + 写 PPT + 写邮件" → 跨领域
- **直接进入 CRISPE 框架** (需要 AI 反问)

### 信号 7: 模糊词占比高
- 包含 "类似"、"什么的"、"等等"、"...那种"
- 用户自己也用模糊词, 说明没想清楚

## 不触发

**以下场景不触发, 直接干活**:

### 场景 1: 用户已给出完整 5W1H
```
"帮我把这段英文合同 (500 字) 翻译成中文, 保留法律术语, 给我做跨境电商的德国客户用, 今天下班前要。"
```
→ 5W 全齐: What (翻译) / Why (跨境电商) / Who (德国客户) / When (今天) / How (保留术语) → **直接干**

### 场景 2: 用户明确说"直接干, 不用问"
- "别问了, 直接做"
- "时间紧, 先做再调"
- "按你的判断来"
→ **直接干**

### 场景 3: 紧急场景
- 系统报错 / 线上事故
- 截止时间 < 1 小时
- 客户催的紧急需求
→ **直接干** (问问题更慢)

### 场景 4: 任务有清晰 SOP
- "按 [模板] 生成 [类型] 报告"
- "跑 [命令] 看结果"
- "执行 [脚本] 然后告诉我"
→ **直接干**

### 场景 5: 重复任务
- 历史对话已做过的同类任务
- 已有明确的执行路径
→ **直接干**

### 场景 6: 闲聊/咨询
- "你觉得 X 怎么样" (闲聊, 不是任务)
- "什么是 Y" (知识咨询)
- 哲学/价值观讨论
→ **不触发, 直接回答**

## 触发阈值配置

### 默认阈值
- 隐式触发: **任意 2 条**信号即触发

### 更激进 (任何模糊都触发)
- 隐式触发: **任意 1 条**信号即触发
- 适合: 用户是新用户 / 重要项目 / 质量优先

### 更保守 (只在明显模糊时触发)
- 隐式触发: **任意 3 条**信号才触发
- 适合: 用户是老手 / 时间紧 / 频繁任务

### 完全关闭隐式触发
- 只用显式触发
- 适合: 用户希望完全控制触发时机

### 配置位置
本文件的"信号清单"和"阈值"可由用户按需调整。修改后 better-act-skill 的 Step 1 检测逻辑会相应变化。

## 快速判断流程

```
用户消息
   │
   ├─ 是显式触发词? ── 是 ──→ 触发
   │
   ├─ 用户说"直接干"/"别问"? ── 是 ──→ 不触发
   │
   ├─ 是紧急/事故/SOP/重复? ── 是 ──→ 不触发
   │
   ├─ 5W1H 都齐? ── 是 ──→ 不触发
   │
   ├─ 隐式信号 ≥ 2 条? ── 是 ──→ 触发
   │
   └─ 默认 ──→ 触发 (安全侧)
```

**默认走安全侧** (触发优先), 避免漏掉真正需要引导的需求。
